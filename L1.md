## Работа с Git и GitHub 

## 1. Что это такое и для чего нужен?

Git - это распределенная система контроля версий нашего кода.
Она нужна чтобы отслеживать изменения, которые происходят со временем. 
Возможности Git:
1. Возможность хранить разные версии проекта.
2. Возможность возвращаться к различным версиям проекта.

## 2. Проверка наличия установленного Git

Верминале выполнить команду git --version.
Если Git установлен, появляется сообщение с версией программы. Иначе будет сообщение о  ошибке.

## 3. Установка 

Загружаем последню версию файа с сайта https://git-scm.com/downloads.
Устанавливаем с настройками по умолчанию.

## 4. Настройка 

При первом использовании необходимо представиться. Для зтого надо ввести в терминал две команды: 
```
git config --global user.name "Ваше имя на английском языке".
git config user.email почта@exemple.com
```
## 5. Синтаксис языка Markdown.

* Жирный текст - (**) с двух сторон
* Курсивный текст - (*) с двух сторон
* Зачеркнутый - (~) с двух сторон
* Выделение заголовков - (#) в начале строки 
* Показать уровень заголовка - (= или -)
* Нумерованные списки - (1,2,3) в начале
* Ненумерованные списки - (*) в начале 
* Вложенные списки - отступ в начале
* Подсветка - (```(на двух абзацах) название языка программирования, текст по середине)


## 6. Команды:
* git init - создание нового репозитория
* git add ... - сохранение файла
* git commit - запись зафиксированных изменений
* git commit -m "Комент" - коментирует прямо из строки тектового редактора
* git commit -a - автоматически индексирует изменения в файл
* git commit -am "Комент"- совмещение коммитов
* git log - просматривает историю
* git log -h - данные команды
* git log --oneline - история в одну строку
* git log --graph - дерево коммитов
* git diff - разница между git
* git status - смортит работу каталога и раздела проиндексированных файлов
* git checkout - возвращение
* git checkout master - переход к актуальному состоянию
* git checkout -b название - создание и переход на созданную ветку
* git switch - заменя checkout
* git switch -s название - создание и переход на созданную ветку
* git reflog - просмотр команд в строке
* git branch - просмотр веток
* git branch (название новой ветки) - создание новой ветки
* git branch -d (название ветки) - создать ветку
* git merge (название ветки) - слияние
* git clone (ссылка на репозиторий) - склонировать внешний репозиторий на ПК
* git pull - скачать все из текущего репозитория и автоматически слиять с нашей версией
* git push - отправить нашу версию репозитория на внешний репозиторий. **Требуется авторизация** на внешнем репозитории
* git remote - покажет псевдоним репозитория
* git remote -v - покажет псевдоним и адрес репозитория
* clear - очистка терминала

## 8. Игнор файлов
Для того,чтобы исключить из отслеживания в репозитории опред файлы и папки, необходимо создать файл ***.gitignore***, и записать в него их названия и шаблоны, соответствующие таким файлам или папкам.

## 9. Создание веток в git
По умолчанию имя основной ветки в Git - **master**.
Создать ветку можно комендой:
```
git branch <имя новой ветки>
```
Список веток в репозитории можно посмотреть набрав команду:
```
git branch
```
Текущая ветка будет отмечена звездочкой: **\*master**

## 10. Работа с удаленными репозиториями

1. Создать акк на GitHub
2. Создать локальный репозиторий
3. Создать удаленный репозиторий
4. Связать локальный репoзиторий с удаленным

Добавить удаленный репозиторий к проекту:
```
git remote add <имя репозитория> <url-адрес репозитория в сети>
```

## 11. Сделать Pull request

* В своём аккаунте на GitHub создать копию репозитория ... с помощью кнопки **"Fork"**.
* Клонировать копию репозитория на локальный компьютер.
* Создать новую ветку.
* Написать нужную инфу
* git remote add <name> <url>
* git push <name>
* git push --set-upstream L1.md new
